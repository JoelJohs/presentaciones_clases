32 vs 64 bits

El procesador maneja X bytes por ciclo de reloj. Eso define la arquitectura.

32 bits (x86 / i686)64 bits (x86_64 / amd64)
Datos por ciclo4 bytes8 bytes
RAM máxima~3.5 GBTerabytes
Ejecuta programas 64bNo—
Ejecuta programas 32b—Sí (modo compatible)
64 bits puede ejecutar 32 bits, pero 32 bits NO puede ejecutar 64 bits.

¿Cómo identificar?

🐧 Linux

lscpu | grep Architecture

x86_64 = 64 bits · i686 = 32 bits

uname -m

Más conciso, mismo resultado

🪟 Windows

Abrir msinfo32:

Inicio → "Información del sistema"

Buscar: "Tipo de sistema"

→ PC basada en x64
→ PC basada en x86

Si arrancas con un live USB de Linux, puedes correr lscpu antes de instalar nada.

Legacy BIOS vs UEFI

Firmware grabado en la motherboard. Arranca la PC.

Legacy BIOSUEFI
Tabla de particionesMBRGPT
Disco máximo2 TB9.4 ZB
Particiones primarias4 máximo128
InterfazSolo textoGráfica + mouse
Secure BootNoSí
Modo del CPU16 bits (modo real)32/64 bits
Problema real: Ventoy no funciona en BIOS legacy antiguas porque su bootloader no las soporta.

MBR vs GPT

Le dice al firmware dónde vive cada partición.

MBR

Master Boot Record — 1983

  • 4 particiones primarias máximo
  • Discos hasta 2 TB
  • El boot code vive en el primer sector (512 bytes)
  • Usado por Legacy BIOS

GPT

GUID Partition Table — 2010

  • 128 particiones primarias
  • Discos hasta 9.4 ZB
  • Redundante (copia al final del disco)
  • Requiere UEFI (o BIOS con parche)

USB Booteable para Legacy

Ventoy falló → dd es la alternativa.

# 1. Identificar el USB (CUIDADO con el dispositivo)
lsblk

# 2. Desmontar si está montado
sudo umount /dev/sdX1

# 3. Escribir ISO directo al dispositivo
sudo dd if=tu-iso.iso of=/dev/sdX bs=4M status=progress

# 4. Sincronizar
sync
⚠️ of=/dev/sdX = dispositivo completo (sin número).
Si pones sda en vez de sdb, borras tu disco principal.
¿Por qué dd sí? Copia bit por bit, MBR incluido. Ventoy instala su propio bootloader y las BIOS viejas no lo reconocen.
